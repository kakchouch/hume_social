# Hume Social Documentation

This folder provides 3 depth levels so different audiences can onboard quickly.

## Documentation Levels

1. [Beginner Guide](/docs/beginner/)
	Audience: Product, PM, new contributors.
	Focus: What the platform does and how to run it.

2. [Intermediate Guide](/docs/intermediate/)
	Audience: Active contributors.
	Focus: Request flows, app boundaries, and implementation patterns.

3. [Expert Reference](/docs/expert/)
	Audience: Maintainers and reviewers.
	Focus: Full inventory of apps, models, views, templates, settings, and operational details.

4. [Object Reference Map](/docs/reference/)
	Audience: Developers needing object-by-object navigation.
	Focus: Linked documentation for classes, parameters, templates, and cross-app relationships.

## Suggested Reading Path

1. Start with [Beginner Guide](/docs/beginner/).
2. Move to [Intermediate Guide](/docs/intermediate/) when you begin coding.
3. Use [Expert Reference](/docs/expert/) as the canonical technical reference.
4. Use [Object Reference Map](/docs/reference/) to navigate directly between linked objects.
