# Hume Social Platform Diagrams

This folder contains visual diagrams explaining the inner workings of the Hume Social platform codebase.

## Diagrams

1. **architecture_overview.md** - High-level architecture showing all major components and their relationships
2. **content_validation_flow.md** - Sequence diagram of content creation and community validation process
3. **user_progression_system.md** - State diagram showing user permission levels and progression
4. **database_relationships.md** - Entity-Relationship diagram of all model relationships

## How to View

These diagrams use Mermaid syntax and can be viewed in:
- GitHub (renders automatically)
- VS Code with Mermaid extension
- Any Markdown viewer that supports Mermaid
- Online Mermaid editors

## Platform Overview

The Hume Social platform is designed for structured intellectual discourse with:
- **Structured Argumentation**: MiniThesis enforces logical structure
- **Community Validation**: Distributed quality control through tagging
- **Progressive Permissions**: Users earn capabilities through competence
- **Personalized Discovery**: Feed algorithm matches content to user interests

For more details, see the main project README.md.
